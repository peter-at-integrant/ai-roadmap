# Goal 9: LLM Fundamentals and Local Model Exploration

**Phase:** Level 1 — AI Integration Engineer
**Owner:** All Engineers
**Target:** Month 2

## Description

Build foundational understanding of how LLMs work, how they differ from agents, and how to run open-source models locally. Every engineer completes a hands-on prototype using a local model and a vector database, building practical capability for evaluating and hosting models internally.

## SMART Breakdown

- **Specific:** Train all engineers on LLM vs. agent distinctions, run at least 2 open-source models locally using Ollama or LM Studio, integrate a vector database (e.g., Chroma), and build one working RAG or CAG prototype per team member.
- **Measurable:** Each engineer submits a documented RAG/CAG demo (code + write-up) to the team knowledge base by end of Month 2.
- **Achievable:** Ollama, LM Studio, and Chroma are free, run on developer machines, and have extensive documentation. No cloud cost or new infrastructure required.
- **Relevant:** Builds internal capability to evaluate and host models without cloud LLM dependency — critical for sensitive data workloads and cost control.
- **Time-bound:** Training sessions complete and prototypes submitted by end of Month 2.

## Deliverables

- [ ] Training session: LLMs vs. agents — concepts, capabilities, limitations
- [ ] Hands-on lab: install Ollama or LM Studio and run 2 open-source models (e.g., Mistral, LLaMA)
- [ ] Hands-on lab: set up Chroma vector DB and create embeddings from a sample dataset
- [ ] Hands-on lab: build a RAG or CAG prototype connecting the local model to the vector DB
- [ ] Each engineer's prototype documented and submitted to team KB
- [ ] Team debrief: share learnings, compare models, discuss trade-offs

## Acceptance Criteria

- Every engineer has run at least 2 open-source models locally.
- Every engineer has a working RAG or CAG prototype submitted to the KB.
- Team can articulate the difference between an LLM, an agent, RAG, and CAG — verified in the debrief session.

---

## Learning Objectives

By the end of this goal, you should be able to:

- Explain the difference between an LLM and an AI agent
- Run a local model with Ollama and compare its output to a cloud model
- Explain what RAG is, what a vector database does, and when you'd use each

---

## Self-Assessment

Answer the following questions **in your own words** without AI assistance. Reviewed by EM/TL.

1. What's the difference between an LLM and an AI agent?
2. When would you use a local model (Ollama) instead of a cloud API? What are the trade-offs?
3. What is RAG? Give a concrete example of when you'd use it on this project.
4. What is an embedding, and why does it enable semantic search?
